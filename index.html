android studio Gradle构建工具

Gradle 概念

Gradle是一个基于Apache Ant和Apache Maven概念的项目自动化建构工具，配置文件的写法按照groovy语言的语法来写。
如果你想要深入了解Groovy的两大基本概念（任务和项目），可以翻看Gradle官方的使用手册第九章以后的内容，如果你难以坚持阅读英文文档，貌似掉线大神翻译了1.12版本文档Gradle中文翻译。

##1. Gradle 项目结构
MyApp
├── build.gradle
├── settings.gradle
└── app
├── build.gradle
├── build
├── libs
└── src
└── main
├── java
│ └── com.package.myapp
└── res
├── drawable
├── layout
└── etc.

###1. %project%/settings.gradle
声明项目里所有的模块

###2. %project%/build.gradle
项目的Gradle编译文件

buildscript方法定义全局的属性

repositories定义了jcenter作为仓库,可以尝试替换为 mavenCentral() dependencies。一个仓库代表着你的依赖包的来源，例如maven仓库。

dependencies用来定义构建过程，这里要添加与android studio版本对应的gradle插件

all projects方法可以用来定义各个模块的公共属性，以后你可以在这里增加的新tasks和全局属性。比如定义全局配置： 

ext {
        compileSdkVersion = 22
}

再从子module里引用这个配置 

` android {
       compileSdkVersion rootProject.ext.compileSdkVersion
       buildToolsVersion rootProject.ext.buildToolsVersion
 } `

###3. %module%/build.gradle
module的Gradle编译脚本

第一行引入的是google开发，用于android项目和依赖的打包、测试android插件，该插件依赖于java插件，而java依赖于base插件，所以android插件依赖了他们的tasks并定义了自己的tasks

assemble: 集合所有的output 

clean: 清除所有的output

check: 执行所有的checks检查，通常是unit测试和instrumentation测试

build: 执行所有的assemble和check `

connectedCheck 在测试机上执行所有测试任务

deviceCheck 执行所有的测试在远程设备上

installDebug和installRelease 在设备上安装一个特殊的版本

所有的install task对应有uninstall 任务

你可以在命令行或者AS自带的终端中找到项目根目录，然后输入指令来执行这些任务（win10的命令行需要改到低版本），不过android studio拥有图形化工具，而且在你更改Gradle文件后也会有相应提示。如：你只需点击工具栏中的Sysc Projects With Gradle Files 即可执行generateDebugSources tasks命令来同步部署项目。

Android Studio 入门（二） 目录结构

1.项目结构

Android Studio 项目结构
Android Studio 的一个 project (项目)下可以包含多个独立的 module(模块)。
类似于Eclipse中的workspace(工作空间)和project(项目)，但是实际上和Eclipse的概念完全不同。Eclipse的workspace中可以有多个”完整”项目（可以启动多个不同App），而AS中project下默认只有一个”完整”的项目，module只是项目的一部分。

2.文件结构

Android Studio项目可以显示为多种结构类型，其中常用的有project和android结构类型

(1) Project结构类型

  • .idea：IntelliJ IDEA文件夹
  • app
    • build 编译后的文件存放目录
      • generated 自动生成的代码资源
      • intermediates 依赖包的编译缓存
      • outputs/apk 存放临时生成的apk
    • libs 第三方库
    • src 源文件文件夹
      • androidTest 有Context的测试类，Android相关测试，数据库、SP等
      • main
        • java 存放项目代码
        • res 资源文件 minpmap文件的引用在旧版本Android也可以正常运行
        • AndroidMainifest.xml 清单文件
      • test 没有Context的测试类，纯java测试，网络、Utils、Bean等
         .gitgonre
         app.iml
         build.gradle
         proguard-rules.pro 混淆文件
  • gradle文件夹
    • wrapper文件夹
      • gradle-wrapper.jar
      • gradle-wrapper.projecties

Android Studio 入门（一） 简介和安装

1.AndroidStudio 介绍

1.1.介绍 Android Studio 是一个Android开发环境，基于IntelliJ IDEA，Google在前年的I/O大会上推出后开始普及。
1.2.为什么要学习AS？
    1.1.1 Eclipse的ADT开发插件已经不再更新
    1.1.2 有些GitHub的开源库只提供AS的工程代码
1.3.与Eclipse的比较
    1.3.1AndroidStudio是Google官方推出的，专门为Android量身订做的一个开发工具，以后肯定会越来越强大;
    1.3.2速度更快(在大项目上体现明显，电脑配置越高越明显);
    1.3.3UI更漂亮(自带了黑色的炫酷主题)
    1.3.4更加智能的提示补全等功能
    1.3.5整合了Gradle构建工具，Gradle集合了Ant和Maven的优点，不管是配置、编译、打包都非常棒
    1.3.6强大的UI编辑器，附带多设备的实时预览
    1.3.7内置终端，即命令行输入(win下要注意WIN10的新版控制台有问题，建议使用旧版)
    1.3.8 目录结构和Eclipse不一样
    1.3.9 拥有强大、方便的插件

2.AndridStudio的安装

2.1 安装包版本介绍
    2.1.1 下载地址：
        官方的下载地址：http://developer.android.com/sdk/index.html
        国内的下载地址：http://www.androiddevtools.cn
    2.1.2 带SDK和不带SDK版
    2.1.3 安装版和解压版
2.2 展示安装过程
    2.2.1 AS的安装路径和工程路径都不要包含中文
2.3 第一次打开需要配置
    2.3.1 在Android Studio安装目录下的 bin 目录下，找到 idea.properties 文件，在文件最后追加 disable.android.first.run=true，避开连接谷歌服务器更新SDK
2.4 可能出现的问题
    没有配置JAVA_HOME 环境变量
