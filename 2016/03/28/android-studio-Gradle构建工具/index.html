引言

Gradle是一个基于Apache Ant和Apache Maven概念的项目自动化建构工具，配置文件使用groovy语言的语法。学习Gradle的一手资料是Gradle User Guide、Gradle用户指南中文翻译，可以看到内容非常多。除了这文档以外网上大多是千篇一律的android IDE中Gradle构建教程，读过之后貌似学会了Gradle的使用，但往往知其然不知其所以然，远远没有到得心应手的程度，一旦需要调整项目结构/写插件提高生产力，我就完全抓瞎了。所以我认为系统的学习还是需要阅读归纳过的专业书籍（官网有推荐）。不过在此之前有必要对目前掌握的基础知识进行归纳，于是就有了这篇Blog。

2. Gradle 生命周期

Gradle构建分为初始化、配置和运行三个阶段。

  1. 初始化阶段，会去读取根工程中setting.gradle中的include信息，决定有哪几个工程加入构建，创建project实例，比如下面有三个工程： include ‘:app’, ‘:lib1’, ‘:lib2’
  2. 配置阶段，会去执行所有工程的build.gradle脚本，配置project对象，一个对象由多个任务组成，此阶段也会去创建、配置task及相关信息。
  3. 运行阶段，根据gradle命令传递过来的task名称，执行相关依赖任务。

2. %project%/build.gradle

项目的Gradle编译脚本

  • repositories

以往我们在使用外部依赖库/工程的时候，往往要从网上下载jar包/源码，然后放到lib/工程根目录下，再通过构建工具建立依赖关系，还要注意包的最新版本变化，每次升级重新导入一遍，非常麻烦。
Gradle支持多种主流的代码仓库，我们只需在Gradle文件中输入依赖包的名字，就可以从云端的仓库里直接获取代码、进行版本更新，在repositories里我们就定义了jcenter作为代码仓库。
在无法下载Jar包的时候，可以尝试替换为mavenCentral() dependencies

  • dependencies

用来定义构建过程 这里应添加与android studio版本对应的gradle插件。

注：Google开发的Gradle插件gradle-wrapper，在项目构建时，AS会启动gradlew.bat（win下）根据Gradle-wrapper.properties里的distributionUrl文件下载对应Gradle插件版本，Gradle的指令前缀由gradle 更改为 gradlew(gradle wrapper缩写)。这样做有3个好处。

  1. 同一项目在不同版本的AS下，使用统一版本的Gradle构建工具，减少构建错误。
  2. 可在未安装Gradle的电脑上使用Gradle构建工具，不必单独安装Gradle。
  3. 升级Gradle可直接将zip包放入下载目录下

    系统盘:\Users\用户名.gradle\wrapper\dists...`

  • all projects方法

可以用来定义各个模块的公共属性，以后你可以在这里增加的新tasks和全局属性，比如使用ext定义变量： 

ext {
      compileSdkVersion = 22
}

再从子module的Gradle文件里引用

android {
      compileSdkVersion rootProject.ext.compileSdkVersion
      buildToolsVersion rootProject.ext.buildToolsVersion
}

3. %module%/build.gradle

module的Gradle编译脚本

  • 第一行引入的是google的android插件，用于android项目和依赖的打包、测试android插件，该插件依赖于java插件，而java依赖于base插件，所以android插件也有以下tasks

    assemble: 集合所有的output

    clean: 清除所有的output

    check: 执行所有的checks检查，通常是unit测试和instrumentation测试

    build: 执行所有的assemble和check `

    connectedCheck 在测试机上执行所有测试任务

    deviceCheck 执行所有的测试在远程设备上

    installDebug和installRelease 在设备上安装一个特殊的版本

    所有的install task对应有uninstall 任务

你可以使用命令行或AS自带的终端，找到项目根目录然后输入指令来执行这些任务，这样不必加入AS的任务队列，速度稍快一些（win10的命令行除外，需要更改到低版本命令行），不过android studio拥有图形化工具+快捷键，而且在你更改Gradle文件后也会有相应提示。如：你只需点击工具栏中的Sysc Projects With Gradle Files 即可执行generateDebugSources tasks命令来同步部署项目。

  • buildTypes
    有的时候，我们会因种种问题而开发不同版本的App（测试/开发、有广告/无广告），在这个闭包下我们可以通过更改appID的方式打包不同版本的项目，但这个方法并不常用，一般会使用。

4. %project%/gradle.properties

在这里可以设置全局配置，比如我们想要加快Gradle构建速度，在这里开启多线程编译

org.gradle.daemon=true
org.gradle.parallel=true
org.gradle.jvmargs=-Xmx1024m

这里的信息也可以从子module中获取
ex： 

//在配置文件添加变量
i = Hello World

然后我们可以从build.gradle添加如下代码，将这个全局配置变量打印到控制台

task printProperties << { println i }
